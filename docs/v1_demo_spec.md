## 1. 概要

Leonardo Jr. v1 は、屋外設置の野生動物検知カメラ (主に熊検知用途) です。
本書は v1 デモ機の現時点の仕様と、TP社 開発チームが動作確認に必要な情報をまとめたものです。

正式マニュアルは Phase 23 (出荷直前) で別途提供します。

## 2. 通知先設定の流れ

### 2.1 アクセス方法

3 通りのアクセス方法があります。本番ではエンドユーザーは方法 (1) のみ使用します。

| 方法 | URL | 用途 |
|------|-----|------|
| (1) QR コード | (本番機の筐体側面に貼付) | 本番運用 |
| (2) LIFF URL | `https://liff.line.me/2009899662-o9okNxR7` | LINE ワンタップ追加が可能 |
| (3) 直接 URL | `https://leonardo-jr-api.onrender.com/setup-v1/{device_id}` | 開発者向け |

### 2.2 PIN 認証

初回アクセス時、または最後の認証から 30 分以上経過した場合、6 桁の PIN を入力する画面が表示されます。

- PIN は本体側面の QR ラベルに記載されています
- デモ機の PIN: `306596` (LJ-671493E4-QDSF)
- PIN 入力に 5 回連続で失敗すると、当該デバイスは 15 分間ロックされます

### 2.3 セッション

PIN 認証成功後、ブラウザに 30 分有効なセッション Cookie が発行されます。

- 同じスマホで 30 分以内に再アクセス → PIN 入力スキップ
- 30 分以上経過 → PIN 再入力要求
- 別のスマホ/別のブラウザ → 各端末で初回 PIN 入力が必要
- シークレットブラウザ → 毎回 PIN 入力 (Cookie が共有されないため)

## 3. 通知先 (subscribers) の管理

### 3.1 対応チャネル

| チャネル | 入力形式 | 備考 |
|----------|----------|------|
| LINE | `U` で始まる 33 文字の userId | LIFF 経由ならワンタップで自動取得・追加可能 |
| メール | `xxx@yyy.zzz` 形式 | 入力検証あり |
| 電話 | `+81xxxxxxxxxx` または `0xxxxxxxxxx` | 日本国内番号のみ |

### 3.2 操作

- **追加**: チャネル選択 → 通知先入力 → ラベル (任意) → 「追加」
- **削除**: 各通知先の「削除」ボタン
- **全削除**: 「全通知先を削除」ボタン (確認ダイアログ 2 回)
- **テスト通知**: 「テスト通知を送信」ボタン
- **LINE ワンタップ追加**: LIFF 経由でアクセスした場合のみ表示

### 3.3 制限

- 同一デバイスに同じ通知先を重複登録不可 (channel + target が一意)
- 通知先の登録上限は現時点で設けていません (将来制限予定)

## 4. 通知の動作仕様

### 4.1 検知時の通知

野生動物が検知されると、登録済みの通知先全員に以下のように通知されます。

| チャネル | タイミング | クールダウン |
|----------|-----------|------------|
| LINE | 即時 | なし |
| メール | 即時 | デバイス単位 120 秒 |
| 電話 | 熊検知時のみ | デバイス単位 300 秒 |

### 4.2 クールダウンの意味

短時間に連続して検知が発生した場合、登録済み通知先全員にメール/電話が殺到するのを防ぐため、デバイス単位でクールダウンを設けています。

例: メール cooldown 120 秒
- 12:00:00 検知 → 全メール通知先に送信
- 12:01:00 検知 → メール送信スキップ (cooldown 中)
- 12:02:00 検知 → メール送信再開

LINE はクールダウンなし、すべての検知で即時通知されます。

### 4.3 テスト通知

「テスト通知を送信」ボタンの挙動:
- LINE / メール: 全通知先に送信
- 電話: **発信しません** (デモ中の混乱回避)

UI に「電話番号は実際の検知時のみ呼ばれます」と表示しています。

## 5. セキュリティ仕様 (デモ機)

| 項目 | 仕様 |
|------|------|
| PIN 認証 | HMAC-SHA256 ステートレス方式、デバイス単位で固定 |
| ブルートフォース対策 | 5 回失敗で 15 分間ロック |
| セッション | JWT (30 分有効、HttpOnly Cookie) |
| 通信 | HTTPS のみ、CORS 制限済み |
| Cookie | HttpOnly, Secure, SameSite=Lax, パス分離 |
| 横デバイスアクセス | JWT 内に device_id を含めて検証 |

### 本番機 (Mark II) でのセキュリティ強化予定

- ユーザーアカウント (email + password + JWT) ベースの認証
- 個体の CPU シリアル番号 + プログラムシリアル番号の二重照合
- LUKS 暗号化 (CPU バウンド鍵導出)
- 分解検知時の自己破壊オプション

## 6. 既知の制限事項 (デモ機)

| 項目 | 内容 | 対応予定 |
|------|------|---------|
| 同時運用デバイス数 | 1 台 (LJ-671493E4-QDSF のみ) | Phase 4 で複数台対応 |
| LIFF URL のデバイス固定 | デモ機 1 台に固定 | Phase 22 で QR 経由動的化 |
| サーバー単一インスタンス | Render Starter プラン | 本番時にスケールアップ |
| 入力検証 | フロントエンドのみ | 本番機ではサーバー側にも実装 |
| デモ機の認証モデル | デバイス単位 PIN | 本番機ではユーザーアカウント |

## 7. トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| PIN 入力で「ロック中です」と出る | 5 回連続失敗した | 15 分待つ |
| メール通知が届かない (バウンス) | アドレス形式が不正 | 入力形式を確認 |
| テスト通知ボタンを押しても電話が鳴らない | デモ中の混乱回避のため意図的にスキップ | 仕様通り |
| LIFF URL 開いた瞬間に PIN 画面に飛ぶ | 30 分セッションが切れている | PIN 再入力で OK |
